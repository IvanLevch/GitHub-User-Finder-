# GitHub-User-Finder
**Автор:** [Левченко Иван]
**Дата** 30.04.2026

## Описание

GitHub User Finder — это графическое приложение на Python для поиска пользователей GitHub с использованием официального API. Программа позволяет:
* искать пользователей GitHub по логину;
* просматривать результаты поиска в удобном интерфейсе;
* добавлять пользователей в избранное;
* сохранять избранных пользователей локально в формате JSON;
* удалять пользователей из избранного.

Приложение использует библиотеку `tkinter` для создания графического интерфейса и `requests` для работы с GitHub API.

## Требования

* Python 3.6 или выше;
* библиотека `requests` (устанавливается через pip: `pip install requests`);
* подключение к интернету для работы с API GitHub;
* операционная система с поддержкой графического интерфейса (для отображения окна приложения).

## Особенности реализации

1. **Работа с GitHub API**
   * используется публичный API GitHub (https://api.github.com/users/{username});
   * анонимные запросы (до 60 запросов в час);
   * обработка HTTP статусов (проверка на 200 OK и ошибки);
   * обработка сетевых ошибок (отсутствие соединения, таймауты).

2. **Локальное хранение данных**
   * избранные пользователи сохраняются в файле `favorites.json`;
   * структура JSON включает: логин, имя, URL аватара, URL профиля;
   * автоматическое сохранение после добавления/удаления;
   * загрузка избранного при запуске приложения.

3. **Графический интерфейс**
   * построен на стандартной библиотеке `tkinter`;
   * интуитивно понятная навигация: поле поиска → результаты → избранное;
   * динамическое обновление кнопок («Добавить в избранное» ↔ «Удалить из избранного»);
   * использование виджетов `ttk` для современного стиля элементов.

4. **Обработка ошибок и валидация**
   * проверка пустого поля поиска;
   * уведомления о не найденном пользователе (404);
   * защита от дублирования в избранном (проверка по логину);
   * сообщения об ошибках сети и файловых операциях.

5. **Архитектура данных**
   * история поиска не сохраняется — только избранное;
   * каждый пользователь в избранном хранится как объект с ключевыми полями;
   * файл `favorites.json` читается/записывается целиком при операциях.

6. **Пользовательский опыт**
   * мгновенная обратная связь: сообщения `messagebox` после действий;
   * визуальное обновление списков без перезагрузки приложения;
   * сохранение состояния избранного между запусками.

7. **Безопасность и надёжность**
   * отсутствие необходимости в токенах для базового поиска;
   * изоляция данных пользователя (локальные файлы);
   * обработка исключений на всех уровнях (сеть, файлы, ввод).

8. **Оптимизации**
   * минимальные запросы к API (1 запрос на поиск);
   * кэширование избранного в памяти во время сессии;
   * ленивая загрузка: данные загружаются только при необходимости.
